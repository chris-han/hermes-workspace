# Semantier Landing Page Content Plan

## Canonical Narrative Reference

This document derives from [`../docs/semantier-narrative-framework.md`](../docs/semantier-narrative-framework.md), the single source of truth for Semantier's category, mission, paradigm, and messaging hierarchy. Changes to those elements must be made in the Narrative Framework first, then propagated here.

## Purpose

Create a landing page for Semantier that introduces a new enterprise technology category: the Semantic Tier—the Certainty Infrastructure for AI-native Enterprises. The page should explain how Semantier turns semantic uncertainty into governed certainty by making agentic actions justified, authorized, replayable, and auditable before execution.

Use `SEMANTIER_VISION_STORYLINE_HERO_QUEST.md` as the narrative spine for the page: future AI-native organization problem, human value anchor, Semantic Tier, governed action, scale challenge, and future work.

The page should not position Semantier as a generic AI assistant, accounting app, Digital CFO, or dashboard. The core message is:

```text
Don’t align the mind.
Govern the action.
```

The page should make one argument clearly:

```text
The prerequisite for enterprise AI to become trustworthy and adopted in mission-critical scenarios is a human value anchor.

Semantier operationalizes that anchor by governing agentic actions through facts, authority, policy, justification, replay, and auditability.
```

### 3. Problem: Plausibility Is Not Permission

Goal: explain the paper’s central distinction in product language and tie it to the human value anchor.

Section title:

```text
Plausible output is not an admissible action.
```

Body:

```text
LLM agents can produce fluent recommendations, balanced journal entries, confident HR decisions, and polished approval notes. In enterprise environments, that is not enough. A proposed action must remain connected to human-defined value, authority, responsibility, and policy before it can execute.
```

Key contrast:

```text
Mind alignment asks: is the model internally aligned with human intent?
Admissibility asks: is this proposed action allowed to happen?
Human value anchoring asks: whose purpose, responsibility, and authority make this action legitimate?
```

Three cards:

```text
Evidence gap
The AI has a plausible answer but insufficient source facts.

Authority gap
The action requires a role, approval threshold, or policy owner that is missing.

Replay gap
The decision cannot be reconstructed later from pinned facts, rules, and context.
```

### 4. Core Model: Governed Action Loop

Goal: introduce Semantier’s operating mechanism.

Section title:

```text
From AI proposal to governed execution.
```

Flow:

```text
1. Evidence intake
2. Fact admission
3. AI expert proposal
4. Justification artifact
5. Authority and policy validation
6. Allow / correct / escalate / block
7. Replay binding
8. Audit-ready record
```

Short explanation:

```text
Semantier separates generation from execution. AI experts can propose actions, but execution is gated by semantic governance. The system can allow low-risk actions, correct recoverable proposals, escalate ambiguous cases, block unsafe actions, and preserve replayable evidence for later audit.
```

Recommended visual:

```text
A horizontal or vertical pipeline with the four governance outcomes highlighted:
ALLOW, CORRECT, ESCALATE, BLOCK
```

### 7. HR: Governance of Organizational Order And Trust

Goal: show expansion beyond finance without overclaiming.

Section title:

```text
HR governs trust as organizational execution capacity.
```

Body:

```text
AI-native organizations cannot govern money without governing the trust structure that makes people effective. A common founder mistake is to treat reliability as something solved by screening: find "reliable people" once and the organization will execute. That assumes people are static. In reality, people change, incentives shift, contexts evolve, and decision quality must be governed over time.

Semantier should frame HR as trust governance, not employee surveillance. The system does not watch every discrete action in real time. It supports trusted decisions and evaluates decision quality from continuous decision trajectories: what evidence was used, which authority was invoked, which policies applied, where exceptions appeared, how corrections were handled, and whether outcomes can be replayed.

This is the operational version of "trust, but verify." HR workflows become the place where role, authority, responsibility, trust, and organizational learning are governed together.
```

Example workflows:

```text
Role and permission change review
Hiring approval and headcount exception handling
Compensation exception routing
Performance-review evidence organization
Policy eligibility checks
Organizational learning and responsibility records
Decision trajectory review for role, authority, and trust decisions
```

Customer value:

```text
Clearer authority boundaries
Trust decisions based on trajectory evidence, not static impressions
Better support for people and role decisions
More consistent exception handling
Stronger link between work, responsibility, learning, and governance
```

### 8. Trust Stack

Goal: introduce the technical trust tier in accessible language.

Section title:

```text
The trust stack behind governed AI work.
```

Stack:

```text
Semantic Tier
Defines facts, concepts, authority hierarchy, and policy meaning.

Justification Gate
Requires each proposed action to cite evidence, rule context, and execution intent.

Authority Layer
Checks whether the right human or organizational authority exists.

Governance Outcomes
Allows, corrects, escalates, or blocks proposed actions.

Replay Binding
Pins facts, rules, artifacts, and context so the decision can be reconstructed later.

Audit Evidence
Preserves the trail required for review, dispute resolution, and external trust.
```

Keep this section concise. Avoid turning the landing page into a paper summary.
